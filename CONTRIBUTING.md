# Contributing to Problem2Project (VIT-Voice)

First off, thank you for considering contributing to VIT-Voice! This platform is built by VIT students, for VIT students. Your contributions help make campus life and academic growth better for everyone.

## Code of Conduct
By participating in this project, you are expected to uphold a welcoming, inclusive, and respectful environment. Be kind to your fellow students and developers.

## How Can I Contribute?

### 1. Reporting Bugs
If you find a bug, please check the existing issues to ensure it hasn't already been reported. If it's a new bug, open an issue using the **Bug Report** template.

### 2. Suggesting Enhancements
Have an idea to make Campus Life Mode or Growth Mode better? Open an issue using the **Feature Request** template. We love new ideas!

### 3. Pull Requests
If you want to write code and fix an issue or add a feature, follow these steps:

#### Git Workflow
1. **Fork** the repository on GitHub.
2. **Clone** your fork locally: `git clone https://github.com/YOUR_USERNAME/problem2project.git`
3. **Add the upstream remote**: `git remote add upstream https://github.com/ORIGINAL_OWNER/problem2project.git`
4. **Create a new branch** for your feature or bugfix (see Branch Naming Conventions).
5. **Commit** your changes (see Commit Message Conventions).
6. **Push** the branch to your fork: `git push origin branch-name`
7. **Open a Pull Request** against the `main` branch of the original repository.

#### Branch Naming Conventions
Use descriptive branch names. Prefix your branch with the type of work being done:
* `feat/` for new features (e.g., `feat/karma-voting`)
* `fix/` for bug fixes (e.g., `fix/header-alignment`)
* `docs/` for documentation changes (e.g., `docs/update-readme`)
* `refactor/` for code refactoring (e.g., `refactor/button-components`)

#### Commit Message Conventions (Conventional Commits)
We follow the [Conventional Commits](https://www.conventionalcommits.org/) specification:
* `feat: add anonymous posting feature`
* `fix: correct layout shift on mobile devices`
* `docs: update contribution guidelines`
* `style: fix linting errors in page.tsx`
* `refactor: extract ThemeProvider to separate file`
* `chore: update dependencies`
